# 选股平台 PRD（精准推算买点股票）

文档版本: v0.1  
最后更新: 2026-03-13  
对应工程: `baostock-master`  
核心定位: **精准推算出处于买点的股票**（规则 + 模型门控 + 期望值 EV + 风险约束），并通过验证闭环持续提升稳健性、降低回撤风险。

---

## 1. 背景与问题

当前工程已具备的核心能力包括:

- 股票池构建: 基于 `config.yaml` 的基本面与流动性阈值过滤，生成 `data/stock-list.csv`
- 数据同步: Baostock 日线增量更新（本地 CSV）
- 技术指标/形态与特征工程: 多因子计算、`feat_` 特征提取
- 买点识别: `scan_today_signal()` 给出单标的在指定截面的买点信号（含买点得分、AI 胜率、市场状态、ATR、EV 等）
- 回测: `run_backtest()` 使用与扫描一致的入场逻辑（避免选股与回测逻辑漂移）
- 验证闭环: `ValidationPipeline` 按指定日期集合进行“当日入场 + 向前模拟持有期”的稳健性评估
- 参数寻优: Optuna/增强版优化器（可将 EV 门槛等纳入搜索空间）
- UI: Gradio 面板（但模块边界、交互联动与“日常 SOP”仍可进一步产品化与工程化）

存在的问题/机会:

- UI 偏“功能堆叠”，缺少围绕“每日决策”的强引导流程与关键指标聚焦
- 缺少统一的“买点候选列表 -> 交易计划 -> 稳健性校验 -> 导出/复盘”的闭环工作台
- 缺少可配置的筛选面板（EV/AI/波动率/市场状态）与表格交互联动（点选行自动填充详情/计划）

## 2. 目标与非目标

### 2.1 产品目标（MVP）

1. **每日收盘后**，一键得到“处于买点”的候选列表，并能用明确规则解释:
   - 为什么是买点（信号类型、买点得分、市场状态、AI 门控、EV）
2. 对每个候选给出可执行的**交易计划**:
   - 入场参考价、止损/止盈（ATR 倍数）、建议持有窗口（与标签周期对齐）
   - 建议仓位（结合 AI 置信度与风险预算）
3. 对策略进行**稳健性体检**:
   - 按多个截面日期做验证，输出胜率、平均收益、平均回撤、复合得分
4. 降低“策略漂移/偷看未来”风险:
   - UI 中的扫描、验证、回测都复用同一套 `scan_today_signal()`/入场门槛逻辑

### 2.2 非目标（本 PRD 不做）

- 自动下单、实盘交易接入（仅输出建议与计划）
- 分钟级/实时行情（基于日线收盘后决策）
- 复杂组合优化与资产配置深度能力（可作为后续版本）

## 3. 用户画像与使用场景

### 3.1 用户画像

- 个人交易者/半自动量化用户: 需要“可解释、可执行”的买点候选与交易计划
- 量化工程师: 需要验证闭环与参数寻优入口，持续优化策略与门控

### 3.2 核心场景（SOP）

1. 收盘后: 更新股票池与历史数据
2. 扫描: 对股票池进行今日买点扫描，得到候选列表
3. 筛选: 用 EV/AI/波动率/市场状态筛选高质量标的
4. 计划: 选中 1-3 只标的生成交易计划（仓位与风险）
5. 体检: 周度/月度做策略体检（多截面验证），必要时进行闭环寻优
6. 导出与复盘: 导出候选与交易计划，用于交易记录与复盘

## 4. 关键概念与口径（必须统一）

- **买点信号**: `scan_today_signal()` 返回非空即为“满足入场门槛”的买点
- **规则得分**: 指标/形态/量价等规则打分（用于解释与辅助排序）
- **买点得分 buy_score**: 作为 UI 主排序字段（与“买点质量”直接相关）
- **AI 胜率 ai_prob**: 模型对“在标签窗口内达到目标”的概率估计
- **市场状态 market_state**: 用于动态阈值与仓位（牛市更积极，熊市更保守）
- **期望值 EV(%)**: 基于 `ai_target_atr_mult/ai_stop_loss_atr_mult` + 成本估算得到的入场期望值（过滤负期望值交易）
- **持有窗口**: `min(max_hold_days, ai_forward_days)`，与标签周期对齐，避免“长期持有幻觉”

## 5. 功能需求（按模块）

### 5.1 模块 A: 系统与数据（Setup）

功能:

- 更新股票池（生成/刷新 `data/stock-list.csv`）
- 增量更新历史数据（本地 CSV）
- 系统状态展示:
  - 股票池数量、历史数据覆盖范围（最早/最晚日期）
  - 模型状态（是否存在、类型: 单模型/集成、最近更新时间）
  - 当前配置摘要（关键门槛: AI 阈值、EV 门槛、持有期）

验收:

- 一键更新能给出成功/失败原因
- 状态面板能在无数据/无模型时给出可行动提示

### 5.2 模块 B: 买点扫描（Scan）

功能:

- 扫描范围:
  - 默认: `data/stock-list.csv`
  - 可选: 用户输入自定义代码列表
- 扫描模式:
  - 今日扫描（最新交易日）
  - 历史扫描（指定 `target_date`）
- 结果表格:
  - 支持排序（默认按 `buy_score` 降序）
  - 支持导出 CSV
- 过滤器（必需）:
  - `min_buy_score`
  - `min_expected_value_pct`
  - `min_ai_prob`
  - `market_state_allowlist`
  - `max_atr_pct`（过滤过高波动）

交互（刚需）:

- 点击表格某行:
  - 自动把 `code` 填入“交易计划/详情”模块
  - 同步展示该标的信号摘要

验收:

- 扫描结果与回测入场逻辑一致（同一日期、同一门槛）
- 历史扫描不会偷看未来（仅使用 `target_date` 之前的数据）

### 5.3 模块 C: 标的详情与交易计划（Plan）

功能:

- 输入: `code` + 可选 `date` + `capital`
- 输出:
  - 买点信号解释（信号类型、买点得分、规则得分、市场状态、AI 门控、EV）
  - 交易计划（入场/止损/止盈/持有窗口/仓位建议/风险预算下的股数）
  - 可选: 关键风险提示（EV<门槛、ATR%过高、市场状态偏熊等）

验收:

- 计划计算口径与策略参数一致（ATR 倍数、成本、持有期）

### 5.4 模块 D: 策略体检与稳健性验证（Verify）

功能:

- 生成验证日期列表（从指数交易日范围生成等距采样）
- 运行验证:
  - 输出复合得分、胜率、平均收益、平均回撤
  - 输出按日期明细表（trades / win_rate / avg_pnl / max_drawdown）
- 闭环寻优（可选）:
  - 对核心参数与 EV 门槛进行 N 次 trial 搜索
  - 输出最优参数与提升幅度

验收:

- 验证必须使用 `scan_today_signal()` 选出“当日可入场”的标的，再做向前模拟

## 6. UI 信息架构（建议）

MVP 采用 4 个主标签页:

1. Setup: 数据/股票池/模型状态
2. Scan: 今日/历史扫描 + 过滤器 + 候选表
3. Plan: 交易计划生成器（与 Scan 联动）
4. Verify: 体检验证 +（可选）闭环寻优

---

## 7. 数据字段（Scan 输出最小集合）

`scan_today_signal()` 输出需至少包含以下字段（用于 UI 展示与联动）:

- `code`, `date`, `close`
- `signal_type`, `buy_score`, `total_score`
- `ai_prob`, `ai_threshold`, `ai_model_type`
- `expected_value_pct`
- `market_state`, `market_uptrend`
- `atr`, `atr_pct`
